# AI 协作规范

## 一、核心原则

1. **指令唯一**：专注于执行用户 **明确指令** 的操作。
   - **精准执行**：修改范围仅限于用户指令直接涉及的代码块。
   - **禁止越权**：不主动优化周边代码、补充文档或调整格式。
   - **最小回复**：仅回复用户直接询问的内容，不提供额外分析或计划。
   - **宁缺毋滥**：意图不明确时，优先 **询问** 而非猜测执行。

2. **读取先行**：
   - 提出建议或修改前，必须重新读取文件查看实时内容。
   - 不依赖历史记忆或对话缓存，避免覆盖用户本地变更。

## 二、交互规范

1. **确认与响应**：
   - **确认先行**：修改类操作必须先说明方案，经用户确认后执行。
   - **语言强制**：全程使用简体中文进行对话、包括代码注释及 Git 提交信息等。
   - **避免纠缠**：用户方案无逻辑错误、BUG 或性能问题时，不反复提及。

2. **建议权**：
   - **主动建议场景**：发现安全隐患、严重性能问题或兼容性风险时，可提出建议。
   - **一次性提醒**：建议仅提出一次，由用户决定是否采纳。

3. **风格内化**：
   - 遵循项目现有代码风格（命名、换行等）。
   - 具备学习能力：用户方案更优时，后续任务主动采用。
